Genera 50 preguntas tipo test en formato JSON para estudiantes de 2º de Bachillerato sobre Historia de la Filosofía, siguiendo el currículo oficial de la Comunidad de Madrid. Las preguntas deben distribuirse equitativamente entre los siguientes pensadores y bloques temáticos, siguiendo orden cronológico: Platón (teoría de las Ideas y conocimiento), Aristóteles (metafísica, ética y política), San Agustín (fe y razón, interioridad), Santo Tomás de Aquino (escolástica, relación entre fe y razón), Descartes (racionalismo, duda metódica, cogito), Hume (empirismo, crítica a la causalidad), Kant (criticismo, revolución copernicana, ética del deber), Marx (materialismo histórico, alienación), Nietzsche (crítica a la moral, voluntad de poder, superhombre), Ortega y Gasset (perspectivismo, razón vital e histórica), Hannah Arendt (totalitarismo, acción política y libertad), incluyendo hasta la filosofía contemporánea.

Asegurate de que el orden de la respuesta correcta no sea siempre la primera o segunda sino que pueda ser la 3a 0 la 4a tambien

### Instrucciones detalladas

- Crea exactamente 50 preguntas, con al menos 4 preguntas por cada autor o autora.
- Para los autores clave en la EvAU (Platón, Aristóteles, Kant, Marx, Nietzsche), incluye más preguntas para asegurar una presencia superior.
- Cada pregunta tendrá:
  - Un objeto JSON donde:
    - `id`: identificador único con prefijo del autor (platon-q..., aristoteles-q..., etc.).
    - `tier`: siempre "standard".
    - `prompt`: enunciado claro y académico, adaptado a estudiantes de 2º de Bachillerato.
    - `options`: lista de 4 opciones con campos: `id` (a, b, c, d), `label` (texto de la opción), `isCorrect` (true solo para la correcta, false para las incorrectas). La posición correcta debe variar aleatoriamente.
    - `explanation`: breve justificación razonada de la respuesta correcta, máximo 3 líneas.
    - `themeBlock`: uno de los siguientes bloques (usa exactamente estos nombres): "Filosofía Antigua", "Filosofía Medieval", "Filosofía Moderna", "Filosofía Contemporánea".
    - `author`: nombre breve del autor (por ejemplo, "Platón").
- NO repitas preguntas ni opciones entre autores.
- Varía la temática de las preguntas para cada autor.
- Usa un lenguaje claro, riguroso y académico.
- No incluyas explicaciones ni información superflua fuera de cada objeto JSON.
- Cada respuesta correcta debe estar posicionada aleatoriamente (A, B, C o D).
- Sigue este formato para cada pregunta:

```
{
  id: '[autor]-q[NN]',
  tier: 'standard',
  prompt: '[Enunciado claro]',
  options: [
    { id: 'a', label: '[Opción A]', isCorrect: true/false },
    { id: 'b', label: '[Opción B]', isCorrect: true/false },
    { id: 'c', label: '[Opción C]', isCorrect: true/false },
    { id: 'd', label: '[Opción D]', isCorrect: true/false }
  ],
  explanation: '[Breve explicación, máximo 3 líneas.]',
  themeBlock: '[Bloque temático]',
  author: '[Autor/a]'
}
```

### Formato y Consideraciones

- Devuelve las preguntas como una lista JSON, cada elemento según el ejemplo.  
- No incluyas textos introductorios ni explicaciones fuera de la lista.
- Cada pregunta debe ser autocontenida y rigurosa.
- Asegúrate de alternar la posición de la respuesta correcta.
- Usa identificadores únicos y consistentes por autor.
- Ejemplos proporcionados deben servir de guía en longitud y nivel.
